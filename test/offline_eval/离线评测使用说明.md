# 离线评测使用说明

## 1. 目的

当前项目的离线评测分成两层：

- 规则评测：检查工具选择、二轮检索、澄清、证据状态、答案模式等是否符合预期
- `ragas` 评测：检查答案忠实度、相关性、上下文召回和上下文精度

这两层的脚本入口分别在：

- [app/tool/run_offline_eval.py](../../app/tool/run_offline_eval.py)
- [app/tool/run_ragas_eval.py](../../app/tool/run_ragas_eval.py)
- [app/tool/score_offline_eval.py](../../app/tool/score_offline_eval.py)

所有默认输出都落在：

- [test/offline_eval](./)


## 2. 三个脚本分别做什么

### 2.1 `run_offline_eval.py`

作用：

- 逐条跑离线 case
- 产出原始回答、轨迹结果、`reranked_docs`、`retrieved_contexts`

输入：

- `*.jsonl` case 文件

输出：

- `*.run.*.jsonl`


### 2.2 `run_ragas_eval.py`

作用：

- 读取 `run_offline_eval.py` 的运行结果
- 用当前配置的评测模型重新计算 `ragas` 指标

输入：

- `*.run.*.jsonl`

输出：

- `*.ragas.*.json`

说明：

- 这个脚本只依赖已有运行结果，不会重跑在线问答主链
- 只要评测模型、`ragas` 逻辑或阈值发生变化，都应该重跑这一步


### 2.3 `score_offline_eval.py`

作用：

- 汇总规则评测结果
- 可选地把 `ragas` 结果并到同一份总报告

输入：

- `*.run.*.jsonl`
- 可选 `*.ragas.*.json`

输出：

- `*.score.*.json` 或手工指定的合并报告路径


## 3. 什么时候需要跑哪一步

### 3.1 只需要重跑 `run_ragas_eval.py`

满足下面任一条件时，通常不用重跑 `run_offline_eval.py`：

- 只改了评测模型，例如把 `qwen3.6-flash` 换成别的文本模型
- 只改了 `ragas` 指标脚本
- 只改了 `ragas` 阈值
- 只想重新生成 `ragas` 报告或合并报告

推荐命令：

```bash
uv run python app/tool/run_ragas_eval.py test/offline_eval/<已有run_result.jsonl>
uv run python app/tool/score_offline_eval.py test/offline_eval/<已有run_result.jsonl> test/offline_eval/<combined_report.json> test/offline_eval/<ragas_report.json>
```


### 3.2 必须从 `run_offline_eval.py` 开始全量重跑

满足下面任一条件时，应该从头重跑：

- 改了查询主链
- 改了检索逻辑、重排逻辑或证据状态逻辑
- 改了最终回答 Prompt 或答案输出格式
- 改了离线 case 本身

推荐命令：

```bash
uv run python app/tool/run_offline_eval.py app/tool/offline_eval_cases.seed50.jsonl
uv run python app/tool/run_ragas_eval.py test/offline_eval/<新run_result.jsonl>
uv run python app/tool/score_offline_eval.py test/offline_eval/<新run_result.jsonl> test/offline_eval/<combined_report.json> test/offline_eval/<ragas_report.json>
```


## 4. 当前默认阈值

`score_offline_eval.py` 当前内置的 `ragas` 默认阈值是：

- `faithfulness >= 0.7`
- `response_relevancy >= 0.75`
- `context_recall >= 0.6`
- `context_precision >= 0.6`

如果某个 case 在输入里单独配置了：

- `min_ragas_faithfulness`
- `min_ragas_response_relevancy`
- `min_ragas_context_recall`
- `min_ragas_context_precision`

则以 case 自己的阈值为准。

## 6. 建议的日常使用方式

如果只是调评测：

1. 复用已有 `run_result.jsonl`
2. 只重跑 `run_ragas_eval.py`
3. 需要总报告时再跑 `score_offline_eval.py`

如果是改查询效果：

1. 先重跑 `run_offline_eval.py`
2. 再跑 `run_ragas_eval.py`
3. 最后生成合并报告

这样可以避免每次都做最慢的全链路重放。
